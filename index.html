Problem 1: Multiples of 3 and 5

If we list all the natural numbers below 10 that are multiples of 3 or 5, we get 3, 5, 6 and 9. The sum of these multiples is 23. Find the sum of all the multiples of 3 or 5 below 1000.

My solution

Given the above limit of 1000, I chose to create a function sum that would loop through each of the natural numbers from 1 to 1000. Using the binary operator ||, the loop checks whether each number is divisible by 3 or 5, then adds it to the total if either is true. This way, numbers that are multiples of 3 and 5 (e.g., 15, 30, 45) are added only once.

			var sum = function(limit) {
				    var total = 0;
				    for (var i = 1; i < limit; i++) {
				        if (i%3 === 0 || i%5 === 0) {
				            total += i;
				        }
				    }
				    return total;
				};

				sum(1000);

Problem 2: Even Fibonacci numbers

Each new term in the Fibonacci sequence is generated by adding the previous two terms. By starting with 1 and 2, the first 10 terms will be: 1, 2, 3, 5, 8, 13, 21, 34, 55, and 89. By considering the terms in the Fibonacci sequence whose values do not exceed four million, find the sum of the even-valued terms.

My solution

Once again, I decided to create a function fibSumOfEvens, rather than use JS to achieve a simple calculation. I provided the first two Fibonacci numbers for simplicity's sake, using a while loop to add to the seq array. The even numbers in that array are then returned to another array, arrayEven, and the sum of these numbers is returned (phew!).

				var seq = [1, 2];
				var i = 1;

				var fibSumOfEvens = function(limit) {
					while (seq[seq.length - 1] < limit) {
						seq.push(seq[i] + seq[i - 1]);
						i++;
					}
				}
    
				var arrayEven = seq.slice(0, seq.length - 1).map(function(num) {
        			if (num % 2 === 0) {
            			return num;
			        } else {
            			return 0;
        			}
    			});
    
    				return arrayEven.reduce(function(a, b) {
        				return a + b;
    				});
				};

				fibSumOfEvens(4000000);

Problem 3: Largest Prime Factor

The prime factors of 13195 are 5, 7, 13 and 29. What is the largest prime factor of the number 600851475143?

My Solution

I broke this down into three unsophisticated functions. The first, Factors, creates an array of factors by seeing what numbers divide evenly into the target number. The second function loops over that array and divides each factor by preceding preceding factors in order to identify composite factors. These composite factors are set to 0 in a new array, Primes. Finally, the last function loops through the array backwards to return to higher, non-0 prime factor. This is convoluted code, and there are certainly shorter codes out there.

			var Factors = function(num) {
    			var factors = [];
			    for (var i = 2; i <= Math.sqrt(num); i++) {
			        if (num % i === 0) {
			            factors.push(i);
			        }
			    }
			    return factors;
			};

			var Primes = function(array) {
			    var primes = [];
			    for (var i = 1; i < array.length; i++) {
			        for (var j = 0; j < i; j++) {
			            if (array[i] % array[j] === 0) {
			                array[i] = 0;
			            }
			        }
			    }
			    return array;
			};

			var Largest = function(array) {
			    for (var i = array.length - 1; i > 0; i--) {
			        if (array[i] > 0) {
			            return array[i];
			        }
			    }
			};

			Largest(Primes(Factors(600851475143)));
